Comments have a language of their own

Edwin Cowart

> A comment in programming is a means of annotating software such that code is more human-readable. For the most parts the machines themselves ignore comments during compilation and interpretation. Generally there is a string of characters which allows each programmer to comment out a sign line of text and a pair of tags which indicated which enable the programmer to comment out a block of text. The syntax of the line and block comment depends on the language, but I will use C as the example.

// line comment /** block comment */ console.log("Hello Word") // <---- this is code

> The amount of commenting that comes with code varies significantly depending on the education, workplace, personal style and mood of the programmer. Comments might facilitate a better understanding of the code or they can act as an excuse to right unparsable code. Comments can also be a means of having fun with the next developer who has the misfortune of fixing or adapting your code. This means that the language of comments is largely technical and highly contextual, but I’d like to explore some dialects of this language which are not so heavy on the technical side. It is important to keep in mind that comments are written by programmers with the intention that either the original or future programmer will find these comments useful or entertaining.

// // Dear maintainer: // // Once you are done trying to 'optimize' this routine, // and have realized what a terrible mistake that was, // please increment the following counter as a warning // to the next guy: // // total_hours_wasted_here = 42 [3] // Abandon all hope, ye who enter here. [3] /** * For the brave souls who get this far: You are the chosen ones, * the valiant knights of programming who toil away, without rest, * fixing our most awful code. To you, true saviors, kings of men, * I say this: never gonna give you up, never gonna let you down, * never gonna run around and desert you. Never gonna make you cry, * never gonna say goodbye. Never gonna tell a lie and hurt you. */ [3] //Dear future me. Please forgive me. //I can't even begin to express how sorry I am. [3]

> In many ways comments are quite similar to the notes annotated in the margins of medieval manuscripts. An complaint like “Now I’ve written the whole thing: for Christ’s sake give me a drink”[5] could apply to either a scribe, a typist, or a programmer. Although one medium is physical and the other digital, there is a shared human experience. Like the scribe or typist, the programmer has a limited capacity for mental exertion and has to break and may go get some drinks with their co-workers. Human hands still have to produce code and spend long hours reading and writing.

// I dedicate all this code, all my work, to my wife, Darlene, who will // have to support me and our three children and the dog once it gets // released into the public. [3] // When I wrote this, only God and I understood what I was doing // Now, God only knows [3] // sometimes I believe compiler ignores all my comments [2] // I am not sure if we need this, but too scared to delete. [4]

> I like to think that with such a mathematical medium, the programmer seeks a way of humanizing the code. That this scarcity of human interaction and sense of seriousness that comes with writing code creates a need on the part of the programmer to be silly or playful. It’s a way of expressing oneself in direct contradiction to the core message of the medium which will be interpreted or compiled for a soulless machine. Certainly stress and frustration play a significant role in why these annotations appear in code. Programming can be an unforgiving endeavor, a tiny mistake can break everything, especially when the codebase is ugly.

// I have to find a better job [4] // Magic. Do not touch. [1] //Happy debugging suckers [1]

> The Beauty of Code can actually give a great insight into why these annotations exist, ““beautiful code” is not an abstract virtue that exists independent of its programmers’ efforts. Rather, beautiful code is really meant to help the programmer be happy and productive. This is the metric I use to evaluate the beauty of a program.”[6] Often code does not conform to this beauty standard because managers and programmers come and go from a place of employment and each of them has their own unique style and understanding of the codebase, the language, and external libraries which results in really ugly code. Programmers then have to either fix or add a feature to interwoven or ugly code which results in a ton of frustration and potential will introduce more bugs. This is certainly another reason why programmers might complain or joke after a long day of debugging.

//This code sucks, you know it and I know it. [1] //Move on and call me an idiot later. [1] // no comments for you [1] // If this code works, it was written by Paul DiLascia. If not, I don't know // who wrote it [1]

> These comments display how the culture & humor of the programmer is built on the ideas prevalent through meme and internet culture, which only exist through the existence of programs. Our as the The Medium is the Message generalizes this idea “This is merely to say that the personal and social consequences of any medium—that is, of any extension of ourselves—result from the new scale that is introduced into our affairs by each extension of ourselves, or by any new technology. “ [7] I find this circularity between the medium and the message of comment to fascinating. There very essence of this dialect of comments is derived from the culture built around the mediums which the message is written in.
